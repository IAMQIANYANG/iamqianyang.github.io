Recipe Qoo

Deploy: https://recipe-qoo.firebaseapp.com/

Source: front-end back-end

Recipe Qoo is a web application designed to help family cooks to store, manage, and share their recipes.

More importantly, it is designed to help solve the ultimate question of life: What to eat? by picking a recipe for you.

Recipe Qoo is currently at its v1 stage. The system randomly picks a recipe for you. Later on, more functions will be added including randomly select a recipe based on some criteria, such as ingredients.

Stacks

Front-end: Thick client with React.js + Redux + Bootstrap, hosted on firebase.

Back-end: RESTful API with Node.js + Express.js, hosted on heroku.

DataBase: MongoDB

Auth: Passport-JWT

Twitter Emotion Spam Detector

https://recipe-qoo.firebaseapp.com/

This is a group project for the course LIBR 559N - Text Analytics. The goal is to analyze and detect emotion spams (tweets assigned with a emotion hashtag but donâ€™t actually express the emotion) on Twitter.

Project involved in crawling data, labelling them, conducting an in-depth analysis, and building a machine learning model to identify the major emotion spams.

My major role was to wrote code to normalize the crawled data, train the labelled data using the Support Vector Machine algorithm to identify future emotion spams (accuracy 88.13%).

ShaReader - A social and interactive interface design for e-book

Click to watch a short intro video (2 mins)

This is a work from a project for the course CPSC 554M - Research Topics in HCI.

We designed and developed ShaReader, a high-fidelity prototype that facilitates annotation sharing in the digital reading environment,

on the basis of our understanding of the potential users and tested it for its usefulness and usability.

Results showed that ShaReader is able to facilitate social annotating with minimal distraction to the users.

It also allows users to smoothly switch between the reading and the annotation viewing tasks.

Key words:

User experience design, User experience research, Prototype
